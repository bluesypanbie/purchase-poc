# 아키파이 매입 서비스 POC 기획 초안

본 문서는 아키파이 매입 서비스 POC 기획 초안에 관한 내용을 다룹니다.

문서의 내용은 Claude Code 또는 AI가 읽게 되는 문서입니다.

해당 POC에서, 웹 부분은 Framer에서 제작할 예정입니다.

다만, Framer의 기능에 한계가 있기 때문에, AI 산출가 및 신청폼은 코드 컴포넌트를 통해 직접 구현한 후 프레이머에 넣는 방식으로 진행합니다.

백엔드는 Supabase로 기민하게 구성해볼까 합니다. AI 관련 API 키 값이나 이런 게 프레이머에 그냥 때려박으면 보안 문제가 있기 때문입니다. 또한 신청폼 내용을 저장하는 것도 Supabase에서 진행하고, 신청폼에서, 매물 이미지도 Supabase에 업로드 되어야겠죠?

### 매입 서비스 사용자 플로우 (웹)

```
매입 서비스 화면 접속
↓
매입 서비스 화면 내 입력 필드에 매물 정보 입력
- 예시 1: 2020년식 Boss BD-2 Blues Driver 대만산 상태 양호, 사용감 다소 있음
- 예시 2: 아이바네즈 TS808 연식 모름, LED 고장
↓
정보 입력 후 '완료' 관련 버튼 선택
↓
입력 필드는 잠기고, 입력 필드 위 쪽에 산출된 예상 매입가 표시
- "예상 매입가: ##,###원±" 이런 식으로 표시 (AI가 산출 해 줌)
- 하단 버튼에는 "매입 신청 계속하기"
↓
'매입 신청 계속하기' 버튼 선택
↓
폼 입력
↓
신청 완료 {+ 자동화 문자 또는 카카오톡 알림톡 전달(아키파이 -> 판매 고객)}
```

### 매입 관련 전체 플로우

```
웹에서 매입 신청
↓
담당자 확인
↓
판매 고객에게 배송지 안내
↓
판매 고객이 물품 배송
↓
물품 수령 및 검수
↓
최종 매입가 산출 및 판매 고객에게 안내
↓
판매 고객 동의 시 정산금 지불 / 판매 고객 미동의 시 반송
```

### 초입 화면 구성

---

- 텍스트 입력 컨테이너 (AI 기반 매입가 산출)
    - 입력란
    - 버튼
        - 다시 생성하기
        - 매입가 산출하기

<aside>

‘입력란’에는 다음과 같은 정보를 사용자가 작성해야 합니다.

- 브랜드 및 모델(꼭 브랜드 + 모델 양식이어야 하는 건 아니지만 권장. 이러한 브랜드 및 모델 정보는 매입가 산출을 위해 필수적인 정보.)
- 생산년도 (있으면 도움되는 정도)
- 제조국 (일부 이펙터 모델의 경우 생산년도에 따라 제조국이 다를 수 있고, 가치에 영향을 미치는 경우가 있기 때문. 있으면 도움되는 정도)
- 상태 (사용감 많음 / 민트급 등 일단은 텍스트로 적을 수 있는 상태 정보. 하자가 있다면 하자에 대한 정보가 될 수 있음.)
- 기타 내용(Claude Code나 다른 AI에게 도움 받아야 함.)
</aside>

### 신청 상세 화면 구성

---

- AI 기반 산출 매입가 금액
- 매입가 산출을 위해 사용자 입력했던 텍스트 내용
    - 금액과 텍스트 내용은 앞 단계에서 작성한 거 그대로 들고오기
    - 금액과 텍스트 내용은 백엔드 상에서 조회 가능해야 함.
- 신청폼
    - 신청자 성함
    - 신청자 휴대전화번호
    - 반송주소
        - 주소
        - 상세 주소
    - 정산계좌
        - 은행 (선택형)
        - 계좌번호
        - 예금주명
    - 매물 사진 (최대 10장)
    - 자유 기재란 (선택 입력)
    - 체크박스
        - 서비스 이용약관 동의
        - 신청한 매물은 1년 이내 해외직구 상품이 아닙니다.
        - (또 다른 동의, 확인 관련 항목)

---

기능 구축을 어떻게 해야할지 고민이 많습니다.

입력폼의 경우 원스톱 서비스 신청 관련으로 프레이머에서 코드 컴포넌트가 있습니다. 여기서 하위 컴포넌트와 로직들을 끌고 올 수 있을 것으로 예상합니다. (새로 만들어야 하는 하위 컴포넌트 등 있습니다.)

전반적으로 기획 내용을 보고 조언과 궁금한 점, 질문을 주세요.

피드백이 있으면 그걸 반영한 후 기획 내용을 정리하면 좋겠습니다.

참고로 자동화 툴 Make의 크레딧이 있긴 합니다. 꼭 Make를 써야 한다는 건 아니고 이것을 썼을 때 효율적이고 빠르게 구현이 된다고 판단되면 사용하라고 조언 주면 됩니다.
